解读：Improving Search Ranking for Maps

原文：Improving Search Ranking for Maps
主题：地图搜索、排序、用户注意力、A/B 测试、booking probability。

一句话总结

Airbnb 发现列表排序的 rank-decay 注意力假设不适用于地图，于是围绕地图空间中的用户注意力重新设计排序与展示策略，包括限制低价值 pin、设置 mini-pin 层级、以及将高预订概率房源移动到地图注意力中心附近。

核心问题

列表搜索中，用户注意力通常从第一名开始递减；但地图中房源是空间散点，没有天然第一名。因此单纯按 booking probability 选出房源后铺在地图上，未必能最大化用户发现高质量房源的概率。

概念解释：CTR 与 rank-decay 注意力假设

CTR 是 Click-Through Rate，中文通常叫点击率，计算方式是 点击次数 / 展示次数。例如某个房源 pin 被展示 1000 次、被点击 50 次，则 CTR 为 5%。在这篇文章里，CTR 用来衡量某个列表排名或地图位置上的房源被用户点击的概率。

Rank-decay 注意力假设 指的是：在列表页中，用户注意力会随着排名从上到下递减。第 1 名通常获得最多曝光和点击，第 2 名次之，越靠后的结果越少被看到和点击。这个假设不是 Transformer 里的 attention，而是搜索排序系统对用户浏览行为的建模。

列表页中的点击/预订机会 ≈ 位置注意力 A(rank) × 房源本身转化概率 P(book | listing, user)
其中 A(1) > A(2) > A(3) > ... > A(n)

因此在列表排序中，如果目标是最大化预订，最自然的策略是把 booking probability 最高的房源放到用户注意力最高的位置，也就是按预订概率从高到低排序。

房源预订概率理想列表位置原因
A8%Rank 1最高转化概率，应获得最高注意力。
B5%Rank 2次高转化概率，对应次高曝光。
C2%Rank 3较低转化概率，放在较低注意力位置。

这篇文章的关键洞察是：地图页没有天然的 Rank 1、Rank 2、Rank 3。房源以 pin 的形式散布在地图上，用户注意力不再主要由列表排名决定，而会受到 pin 是否展示、pin 样式、屏幕坐标、是否靠近地图中心等因素影响。所以列表页的 rank-decay 假设在地图页失效，排序问题也从“给房源安排列表位置”变成了“决定哪些 pin 展示、用什么视觉层级展示、以及如何设置地图中心”。

三种注意力建模

策略假设方法结果
Uniform attention展示出的 map pins 近似共享用户有限注意力。限制 pin 集合中最高/最低 booking probability 比值。带来 Airbnb ranking 历史中较大的 bookings improvement，并提升 5-star trips。
Tiered attention不同视觉样式获得不同点击概率。高概率房源显示带价格 oval pins，较低概率房源显示 mini-pins。mini-pins CTR 约为普通 pins 的 1/8，适合桌面固定 pin 数量场景。
Discounted attention地图坐标位置影响注意力。根据坐标 CTR 选择新地图中心，让高 booking probability 房源更靠近中心。A/B 提升 uncancelled bookings 0.27%，map moves 减少 1.5%。

关键启发

  1. 排序目标必须适配 UI 形态：列表、网格、地图不是同一个 ranking problem。
  2. 用户注意力是稀缺资源，展示更多候选不一定更好。
  3. 视觉层级本身就是排序系统的一部分，mini-pin 是把模型分数转化为注意力分配的产品机制。
