解读：Embedding-Based Retrieval for Airbnb Search

原文：Embedding-Based Retrieval for Airbnb Search
主题：搜索召回、Two-tower、Contrastive Learning、ANN、IVF。

一句话总结

Airbnb 构建首个 embedding-based retrieval 搜索召回系统，用 two-tower 模型把 query 和 listing 映射到向量空间，并选择更适合高频更新与过滤条件的 IVF 方案服务线上检索。

业务问题

Airbnb 搜索面对数百万房源，大范围地点和 flexible dates 会产生巨大候选集。需要在重排序模型之前快速筛出相关候选，否则在线延迟和计算成本不可控。

核心技术

模块做法关键原因
训练数据按 trip 聚合历史 query；最终预订房源为 positive；曝光/点击/wishlist 但未预订为 hard negatives。随机负样本太简单，无法学习真实选择边界。
Contrastive learning拉近 query 与 booked listing，推远 query 与 negative listing。学习 query-aware 的房源相关性。
Two-towerlisting tower 处理房源特征，query tower 处理地点、人数、住宿时长等。listing embeddings 可每日离线预计算，线上只算 query tower。
ANN 选择比较 HNSW 和 IVF，最终选择 IVF。HNSW recall 略好，但面对高频价格/可订更新和地理 filter 延迟差；IVF 更易集成现有搜索系统。
相似度函数比较 dot product 与 Euclidean distance。Euclidean 产生更均衡 cluster size，提升 IVF 判别力。

结果

EBR 上线 Search 和 Email Marketing 后，A/B 测试带来 statistically significant overall bookings gain，且提升幅度可与 Airbnb 过去两年最大级别搜索 ranking ML 改进相当。

关键启发

  1. 大规模搜索要分层：召回阶段用高效向量检索，后续再用重排序模型。
  2. 生产 ANN 选择不能只看 recall，还要看更新频率、过滤、内存、延迟和架构集成。
  3. Hard negatives 是检索模型质量的核心，因为它们来自用户真实考虑但未选择的房源。
